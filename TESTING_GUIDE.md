# دليل اختبار النظام بعد التحديث

## نظرة عامة
هذا الدليل يوضح كيفية اختبار جميع الميزات الجديدة والتحسينات التي تمت إضافتها للنظام.

## 🚀 بدء تشغيل النظام

### 1. تشغيل الخادم
```bash
cd D:\hakim\dev\SupabaseService
npm install
npm run dev
```

### 2. تشغيل العميل
```bash
# في terminal منفصل
npm run client
```

### 3. التحقق من الاتصال
- انتقل إلى: `http://localhost:5173`
- تأكد من تحميل الصفحة الرئيسية بنجاح

## 🧪 اختبار APIs الجديدة

### اختبار endpoints إدارة النظام:

```bash
# 1. صحة النظام
curl http://localhost:3001/api/system/health

# 2. إحصائيات قاعدة البيانات
curl http://localhost:3001/api/system/database-stats

# 3. إحصائيات API
curl http://localhost:3001/api/system/api-stats

# 4. نشاط المستخدمين
curl http://localhost:3001/api/system/user-activity

# 5. مهام الصيانة
curl http://localhost:3001/api/system/maintenance-tasks
```

### النتائج المتوقعة:
كل endpoint يجب أن يعيد JSON صحيح مع بيانات حقيقية.

## 📱 اختبار الواجهات المحدثة

### 1. صفحة طلبات الصيانة
**المسار:** `/service-requests`

**اختبارات:**
- ✅ تحقق من تحميل قطع الغيار من `/api/spare-parts`
- ✅ تأكد من عدم وجود بيانات تجريبية
- ✅ اختبر إضافة طلب صيانة جديد

**الخطوات:**
1. انقر على "إضافة طلب صيانة"
2. املأ النموذج
3. في قسم "قطع الغيار"، تأكد من ظهور قطع الغيار الحقيقية
4. احفظ الطلب وتأكد من النجاح

### 2. صفحة الإدارة الموحدة  
**المسار:** `/unified-management`

**اختبارات:**
- ✅ تحقق من التحديث التلقائي كل 30 ثانية
- ✅ تأكد من عرض البيانات الحقيقية للنظام
- ✅ اختبر الإشعارات الذكية

**الخطوات:**
1. افتح الصفحة وراقب الإحصائيات
2. انتظر 30 ثانية وتأكد من تحديث البيانات
3. تحقق من وجود إشعارات بناءً على حالة النظام

### 3. صفحة إدارة النظام
**المسار:** `/system-administration`

**اختبارات:**
- ✅ تحقق من 12 تبويب مختلف
- ✅ اختبر وظائف التصدير الحقيقية
- ✅ تأكد من الإشعارات الذكية

**الخطوات:**
1. **تبويب لوحة التحكم:** تحقق من الإحصائيات الحقيقية
2. **تبويب البيانات:** اختبر تصدير البيانات
3. **تبويب الأمان:** اختبر تصدير سجل المراجعة
4. **تبويب الشبكة:** اختبر تصدير تقارير الشبكة
5. **تبويب السجلات:** اختبر تصدير سجلات النظام

### 4. صفحة الأنشطة
**المسار:** `/activities`

**اختبارات:**
- ✅ اختبر وظيفة التصدير الحقيقية
- ✅ تأكد من عرض الأنشطة الفعلية

**الخطوات:**
1. افتح صفحة الأنشطة
2. انقر على "تصدير السجل"
3. تأكد من تحميل ملف CSV يحتوي على الأنشطة الحقيقية

### 5. صفحة التقارير
**المسار:** `/reports`

**اختبارات:**
- ✅ اختبر تصدير تقارير مختلفة
- ✅ تأكد من تصفية البيانات بالتواريخ

**الخطوات:**
1. اختر نوع تقرير (طلبات الصيانة مثلاً)
2. حدد فترة زمنية
3. انقر "تصدير CSV" وتأكد من التحميل

### 6. صفحة اختبار قطع الغيار
**المسار:** `/test-spare-parts`

**اختبارات:**
- ✅ تحقق من تحميل المخازن من `/api/warehouses`
- ✅ اختبر إضافة قطعة غيار جديدة

## ⚠️ حل المشاكل الشائعة

### 1. خطأ في تحميل البيانات:
**الأعراض:** صفحات فارغة أو رسائل خطأ
**الحل:**
```bash
# تحقق من حالة الخادم
curl http://localhost:3001/api/health

# أعد تشغيل الخادم
npm run dev
```

### 2. عدم ظهور الإشعارات:
**الأعراض:** لا توجد إشعارات في النظام
**الحل:**
- تأكد من تشغيل الخادم
- تحقق من console للأخطاء JavaScript
- أعد تحميل الصفحة

### 3. فشل التصدير:
**الأعراض:** عدم تحميل الملفات
**الحل:**
- تحقق من إعدادات المتصفح للتحميلات
- تأكد من وجود البيانات في قاعدة البيانات

### 4. خطأ 404 في APIs:
**الأعراض:** `404 Not Found` في طلبات API
**الحل:**
- تأكد من تشغيل الخادم على المنفذ 3001
- تحقق من ملف `routes.ts`
